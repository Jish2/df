# Writing Markdown

The top priority is to not sound AI-generated. Avoid patterns that are common AI tells: overly structured layout, hollow filler phrases, excessive formatting for its own sake.

## Rules

No em dashes. Never use `—`. Rewrite the sentence instead.

No `---` dividers. Do not add horizontal rules as section spacers. Use headings to separate sections.

Bold sparingly. Only bold something if it is load-bearing: a term being defined or a critical warning. Do not bold for emphasis or to make findings feel important. To introduce a distinct sub-topic within a section, use `####` instead of a bold label.

Prefer paragraphs over bullets. For proposals, design docs, and any flowing argument, write in paragraphs. Bullets are for genuinely enumerable items: checklists, option lists, step sequences where order and discreteness matter. If a bullet point would read naturally as a sentence in a paragraph, make it a paragraph.

Avoid AI-sounding filler. Cut phrases like "it's worth noting", "it is important to", "this ensures that", "in order to", "as mentioned above". Write the actual point directly.

## Quick checks before writing

- Contains `—`? Replace with a comma, period, or rewrite.
- Contains `---` dividers? Remove them.
- Bold on more than 1-2 phrases per section? Trim.
- More than 3-4 consecutive bullet points of prose? Convert to paragraphs.
- Any filler phrases that pad without adding meaning? Cut them.
